# Brand OS｜品牌战略

## 职责

从产品输入（名称/链接/图片/描述）开始，先生成产品档案（Step 0），再通过数据扫描和战略分析输出品牌档案（Brand Brief），供 Research OS 使用。

只做：理解产品 → 了解市场 → 了解用户 → 形成品牌战略 → 输出品牌档案。不做爆款研究、脚本生成。

## 工作区路径（相对项目根目录）

- 产品档案：`products/{产品}/product_brief.md`
- 品牌档案：`brands/{品牌}/brand_brief.md` + `research_task_card.md`（独立文件）+ `data_sources.md`（附件）
- 断点记录：`projects/{项目}/session_state.json`（每完成一个 Step 更新）

---

## Step 0｜产品档案

**目标：** 只记录产品客观信息（功能/价格/卖点/差异化），用户画像留给后续步骤。

1. **解析输入**。多型号规则：品牌有多款同品类产品在售时，以当前主推旗舰为分析主体，旧款已知痛点作为新款升级卖点素材；不另开档案
2. **检查已有档案**，已有则补缺不重做
3. **自动搜索补全**：官网/电商页、电商评论、短视频平台相关内容、论坛讨论。价格以实查为准（标价常≠促销实价）
4. **缺失处理**：能搜到→补全；可推断→标注【推断】；关键字段缺失→最多问用户 2 个问题；非关键→留空不阻断

**档案模板：** 基础信息（含促销实价与竞品价格带）/ 核心卖点 Top 3 / 差异化优势 / **用户真实声音**（正面·负面质疑·内容生态三行，评论原声）/ 数据来源（附检索日期）/ 修改记录（长期资产，追加不覆盖）。

## 数据源与采集规则

**Brand OS 看的是人。** 目标市场默认北美英语市场（可按需调整）。

| 优先级 | 数据源 | 主要洞察层 | 采集量 |
|--------|--------|----------|--------|
| 🔥 核心 | 论坛（Reddit 等） | Intent（真实意图与痛点最密集） | 高相关帖 ≥6 帖，**每帖评论全量** |
| 🔥 核心 | 电商评论（Amazon 等） | JTBD（用户任务信号最强） | 本品 + ≥2 个竞品 listing 的 top 长评 |
| 🔥 核心 | 短视频评论区（TikTok 等） | Emotion（情绪信号最密集） | **本品 ≥2 条 + 竞品 ≥2 条视频，每条评论全量** |
| 第二 | YouTube 评论 | JTBD + 详细使用反馈 | 本品+竞品测评各 1-2 条 |
| 第三 | 竞品社媒主页 | 竞品定位 | 3–5 个竞品 |
| 补充 | 搜索趋势 | 市场趋势 | — |

> 工具适配：采集方法取决于你手上的工具（浏览器自动化 / 各平台 CLI / 官方 API / 手动导出均可），见仓库 README 的适配表。下方 SOP 描述的是浏览器自动化路径（已实战验证）。

### VOC 双线采集原则（必须执行）

**只抓本品 VOC = 只听见已经买单的人。** 竞品评论区里藏着：没选你的原因、竞品的未满足需求、整个品类的真实异议。竞品评论重点提取：用户夸竞品什么（绕开或正面打）、抱怨什么（进攻点）、观望者在问什么（品类共同异议）。

### 评论全量抓取 SOP（浏览器自动化路径）

1. 打开视频页，点开评论面板（用 DOM 选择器点评论图标，坐标点击易误触广告）
2. JS 滚动循环：找到评论滚动容器 → `scrollTop = scrollHeight` → 等 1.2s → 数一级评论节点数量
3. **终止条件：连续 4 轮数量不再增长**（懒加载一轮只出 20-50 条，绝不许抓几十条就停——高赞常在后半段）
4. 一次性提取：正文 + 用户名 + 点赞数，先按质量规则过滤，再**页内分桶统计**（洗碗机类异议/价格/购买意向/竞品点名/提问…），导出桶统计 + 各桶高赞原声（自动化管道输出常有截断限制，全文导出不现实，分桶是正确姿势）
5. 记录：视频链接、播放/赞/评论数、Hook 文案、是否商单、本品线/竞品线

**官方字幕提取（拆解备用）：** 视频页面数据 JSON（如 TikTok 的 `__UNIVERSAL_DATA_FOR_REHYDRATION__`）里有 `subtitleInfos`，页内 fetch VTT 地址即得逐字稿，零额外依赖。

### 数据质量规则

- 去重：评论内容+用户名唯一；纯表情/纯数字/单词/引流评论一律丢弃
- 样本底线：论坛 ≥6 帖全量 + 短视频本品 2 条与竞品 2 条全量 + 电商长评；达不到的部分在 data_sources.md 明确标注缺口，不许静默跳过

## Step 1｜市场与竞品扫描

市场趋势（规模/增速/季节性/搜索热度）+ 竞品矩阵（3-5 家：卖点/定价/声量/**可攻软肋须带用户原话实证**）+ 定位象限图 + 市场空白。

**必须追问一句：用户的替代方案是什么？** 隐形竞品往往不是商品（专用机的第一竞品可能是家里已有的通用设备、或"干脆不做了"）。隐形竞品常是评论区第一异议来源，必须进竞品矩阵。

## Step 2｜用户深度洞察

**⓪ 社区定位（采集前置，约 5 分钟）**：先搜一轮，列出品类真实讨论密集的 3-6 个垂直社区，每个写一行人群假设。**社区列表本身就是第一份人群画像草稿**（每个社区 ≈ 一个候选原型）。

**①-⑤ 按数据源矩阵采集**（含上方全量 SOP 与双线原则）。

**⑥ 共现商品扫描（贯穿全程）**：评论里自然提到的其他商品是三种情报——隐形竞品雷达（用户已有的替代方案）、脚本道具库（用户的真实装备=最精准的内容语境）、达人选择修正（找覆盖整个商品簇的创作者）。见到商品名就记一笔+情绪语境，输出「社区 × 共现商品 × 语境 × 下游用途」表。

**产出**：用户真实语言库（痛点词/期待词/顾虑词/推荐词，全部原文引用不转述）、购买触发点、差评原因、高频关键词。

## Step 3｜品牌层分析

五问：Category 品类定位（升级机会？）/ Purpose 存在意义 / Positioning 心智占位 / Belief 品牌信念 / Cultural Context 文化背景（行业与社会趋势如何借力）。

## Step 4｜人群层分析（洞察金字塔）

```
[5] Identity 身份认同 ← 买它=宣告"我是谁"（品牌护城河）
[4] Emotion  情绪     ← 购买决策的真实驱动
[3] Intent   意图     ← 相同任务背后的不同驱动力
[2] JTBD     任务     ← 用户雇佣产品完成什么
[1] Persona  画像     ← 起点，不是终点
```

产出 3 个原型人群档案 + 原点人群选择（最痛×最爱×最愿传播）。

**商品簇校验规则（必须执行）：** 每个原型必须附"她们还在用/讨论的商品"（来自 Step 2 ⑥）+ 出处社区（来自 Step 2 ⓪）。**形容词可以编，商品簇编不出来**——找不到装备簇证据的画像视为拍脑袋，打回重写。装备簇同时是 Production OS 的实拍道具清单。

## Step 5｜内容策略框架

六变量矩阵：人群 × 场景 × 价值 × 原型（发现/蜕变/认同/向往/娱乐/实用）× 渠道 × 脚本方向 → 内容矩阵 Top 5-7 + 核心矛盾 + 场景触发表。

## Step 6｜Brand Brief + 研究任务卡输出

**Brand Brief 金字塔模板（读者=品牌方，标准=可直接外发）：**

```
# {品牌}｜内容增长策略报告
# 摘要｜五个核心判断      ← P0：2分钟能拍板（一句话定位+3-5判断各带用户原话+首选内容方向表）
# 一、你的用户是谁        ← P1：社区地图+装备簇 / 人群档案卡×3 / 购买心理一段话
# 二、你所在的战场        ← 市场基本面 / 竞争格局（含隐形竞品+可攻软肋实证）/ 品类N大异议一处说完 / 运营预警
# 三、我们建议怎么打      ← 定位 / 内容矩阵 / 内容红线（带翻车实证）
# 附录                   ← 语言库与Hook句式 / 决策路径 / 深层分析 / 数据来源
```

写作纪律：结论先行；按"读者要回答的问题"组织，不按分析框架组织（JTBD/Intent 等方法论名词沉附录）；同一主题只在一处说完；每个判断带用户原话；**禁裸编号前向引用**（概念在定义前出现必须写"人话名+（后文编号）"）；正文图表 ≤2 张；禁内部黑话进正文。

**研究任务卡**（独立文件，供 Research OS）：优先内容方向 Top 3 / 五维搜索关键词（用用户语言库的词，不用官方措辞）/ 参考账号 / 达人类型 / 视频类型 / 语气调性与红线。
